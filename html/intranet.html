Mise en place d’un Antivirus au sein du parc informatique

  1. Accueil
  2. Portfolio Pro
  3. interface web interne

Contexte

Un annuaire d’authentification LDAP est utilisé au sein de l’entreprise pour des services interne à celle-ci. L’authentification au wifi se faisait en revanche toujours pas un mot de passe. Aucune plateforme centralisant tous les services de l’entreprise n’existait alors.

Objectif

L’objectif était de créer une interface web accessible via les identifiants et mots de passe LDAP de chaque collaborateurs de l’entreprise, modifier leur mot de passe ce qui leur permettrait par la suite après une mise en place du serveur radius et du switch comprenant les différents Vlan, de s’authentifié au réseau wifi interne via leurs identifiants LDAP. Il s’est avéré par la suite que beaucoup de processus pouvait être amélioré graçe à la création de cette interface au sein de l’entrerpise :

  1. centralisation des procédures d’arrivé et de départ des collaborateurs (création de compte, ajout à certain groupes…) fait via un espace administrateur au sein de ce site
  2. unification des mots de passe collaborateurs
  3. création des voucher permettant la connexion au réseau interne pour les invités.

Réalisation

Le choix de réaliser cette interface web à l’aide du Framework Symfony 4 a été fait assez vite pour un soucis de sécurité et de norme(MVC) notamment. Pour travailler j’ai du reproduire la forme de l’annuaire LDAP présent sur le serveur en production afin de ne pas l’altérer par mes différents tests. J’ai donc mit en place une VM centos 7 de même version que le serveur de production, puis j’y ai installé l’annuaire LDAP avec les mêmes configurations que celui de l’entreprise.

Au fur et à mesure un problème de sécurité est ressorti, il a donc était décidé de mettre un système pour contrer le brutforce en place à l’aide d’un serveur Redis pour protéger le site web. Toute la configuration du nombre d’essais pour être ban et du temp de bannissement est configurable via les fichiers de configuration. Par la suite j’ai du travailler avec des API pour administrer les différents compte slack, Gmail. J’ai donc réalisé plusieurs scripts en python pour Gmail via l’api Admin Gsuite SDK.

Il m’a aussi fallu utilisé l’API pour communiqué avec le controller des différentes bornes wifi, cela m’a permit de mettre en place un formulaire au sein du site web pour créer des voucher invité avec temps d’expiration et nombres d’utilisation définit par les informations du formulaire.

Outils utilisés

J’ai utilisé pour ce projet PhpStorm comme IDE, Git pour le versioning et plus particulièrement le Gitlab interne à l’entreprise.

Langage utilisés : HTML, CSS, Javascript, Symfony 4, Python.

Base de donnée : Sqlite.
